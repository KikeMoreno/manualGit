INSTRUCCIONES DE COMANDOS

A. CREACIÓN DE REPOSITORIOS

Existen dos formas de crear un proyecto con Git:
·Ir al directorio del proyecto existente y utilizar la instrucción
$ git init [nombre_proyecto]
Esto crea un nuevo subdirectorio que contiene todos los archivos necesarios del repositorio Git.
· Clonar un repositorio Git existente desde otro servidor con la instrucción:
$ git clone [url]

B. INSPECCIONAR EL REPOSITORIO

Para comprobar el estado de la rama y si existen archivos nuevos o modificados en los que hay que hacer un commit
$ git status
Para listar el historial de commits del proyecto
$ git log

C. REALIZAR CAMBIOS

Para registrar los cambios y añadir un archivo al repositorio local de Git:
$ git add [archivo]
Una vez realizado un git add podemos guardar los cambios con un commit, pero este archivo aun no estará en el repositorio remoto
$ git commit -m "[descripción_del_commit]"

D. CAMBIOS DE RAMAS

Cuando hablamos de ramificaciones, significa que uno toma la rama principal de desarrollo del proyecto, a partir de ahí se ha continuado trabajando en diferentes direcciones para conseguir diferentes versiones o avanzar el proyecto en paralelo para y así de una manera más rápida.
Para crear una nueva rama:
$git branch [nombre_rama]
Para visualizar una lista de las ramas locales en el repositorio:
$ git branch
Otro uso de la instrucción checkout es el de cambiar a una rama específica y actualizar el directorio de trabajo:
$ git checkout [nombre_rama]
