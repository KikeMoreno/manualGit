INSTRUCCIONES DE COMANDOS

Un resumen de las principales instrucciones de comandos de git.

A. CREACIÓN DE REPOSITORIOS

Existen dos formas de crear un proyecto con Git:
·Ir al directorio del proyecto existente y utilizar la instrucción
$ git init [nombre_proyecto]
Esto crea un nuevo subdirectorio que contiene todos los archivos necesarios del repositorio Git.
· Clonar un repositorio Git existente desde otro servidor con la instrucción:
$ git clone [url]

B. INSPECCIONAR EL REPOSITORIO

Para comprobar el estado de la rama y si existen archivos nuevos o modificados en los que hay que hacer un commit
$ git status
Para listar el historial de commits del proyecto
$ git log

C. REALIZAR CAMBIOS

Para registrar los cambios y añadir un archivo al repositorio local de Git:
$ git add [archivo]
Una vez realizado un git add podemos guardar los cambios con un commit, pero este archivo aun no estará en el repositorio remoto
$ git commit -m "[descripción_del_commit]"
Para eliminar los archivos del directorio de trabajo:
$ git rm [archivo]

D. CAMBIOS DE RAMAS

Cuando hablamos de ramificaciones, significa que uno toma la rama principal de desarrollo del proyecto, a partir de ahí se ha continuado trabajando en diferentes direcciones para conseguir diferentes versiones o avanzar el proyecto en paralelo para y así de una manera más rápida.
Para crear una nueva rama:
$ git branch [nombre_rama]
Para visualizar una lista de las ramas locales en el repositorio:
$ git branch
Otro uso de la instrucción checkout es el de cambiar a una rama específica y actualizar el directorio de trabajo:
$ git checkout [nombre_rama]

E. SINCRONIZAR CAMBIOS

Para recuperar todos los datos del proyecto remoto. Después de utilizar este comando, deberiamos tener referencias a todas las ramas del repositorio remoto, que podremos unir o inspeccionar.
$ git fetch
Para integrar los cambios a la rama, realiza una fusión a tres bandas entre las dos últimas instantáneas de cada rama y el ancestro común a ambas creando un nuevo snapshot y su commit correspondiente. Esta operación no es destructiva ya que no se pierde ningún commit al ejecutarla.
$ git merge
Otra forma de integrar los cambios de una rama a otra es cogiendo los cambios de una rama y reaplicarlos sobre otra, lo que viene a ser reorganizar el proyecto. Esta instrucción hace que Git vaya al punto donde tú estás y donde quieres reorganizar, saca las diferencias introducidas por cada configuración de la rama, guarda las diferencias en archivos temporales, reinicia la rama actual hasta llevarla a la misma confirmación de la rama de donde quieres reorganizar y vuelve a aplicar los cambios. Es una forma de obtener un proyecto más lineal y con menos ramificaciones.
$ git rebase
Para enviar tu proyecto a un repositorio remoto:
$ git push
Para recuperar y unir automáticamente la rama remota con tu rama actual.
$ git pull
