Definición Git y GitHub

¿Qué es GIT?

Git es un software de control de código fuente o scm (Source Code Management). En otras palabras, una herramienta que resuelve una serie de problemas a los que han de trabajar con código fuente. Modela los datos más como un conjunto de instantáneas de un mini sistema de archivos. Cada vez que confirmas un cambio, es decir, realizas un commit, éste guarda el estado del proyecto como una foto de todos los archivos en ese momento, y guarda una referencia a esa instantánea. Si dichos archivos no han sido modificados, Git sólo almacena un enlace al archivo anterior idéntico que ya hay almacenado.

¿Qué es GITHUB?

Básicamente, GitHub es una web donde nos proporcionan espacio para almacenar nuestros repositorios utilizando el sistema de control de versiones Git. Proporciona un servicio de servidor en el que podremos subir nuestro repositorio local de Git para poder así compartirlo con otros usuarios en la red.

Los tres estados

· Estados

Git tiene tres estados principales en los que se pueden encontrar nuestros archivos: confirmado confirmado (committed), modificado (modified), y preparado (staged).
- Modificado: los datos han sido modificados pero todavía no están confirmados.
- Preparado: el archivo modificado está marcado para que vaya en la próxima confirmación. Es decir, al realizar un commit, dicho archivo o archivos serán confirmados y guardados en el repositorio de Git.
- Confirmado: los datos están almacenados de manera segura en la base de datos local.

· Secciones

En consecuencia a los tres estados en los que se puede encontrar un archivo, encontramos que en Git, pueden haber tres secciones:
- Directorio local (Working directory): es una copia de una versión del proyecto, sobre los que el usuario tiene acceso para usar y modificar.
- Área de preparación (Staging area): es un estado en el que se almacena información acerca de lo que irá en la próxima confirmación o commit.
- Directorio de Git (Git directory): es donde Git almacena los metadatos y la base de datos del proyeto.

· Proceso

· Primeramente iniciamos el proyecto con Git mediante el comando git init. Con ello, haremos que nuestro directorio use las funcionalidades de Git.
· En segundo lugar, crearemos nuestros archivos en dicha carpeta o directorio. En este caso, estaremos todavía en el Working directory. Una vez realizado git add, los archivos seleccionados se prepararán para ir a la Área de preparación.
· Finalmente, cuando ya tengamos un número suficientes de modificaciones y hayamos preparado todos los datos, podremos confirmarlos mediante el uso de git commit. Obtendremos el resultado de que estos archivos han sido añadidos al repositorio o directorio de Git, donde serán almacenados y tendermos nuestra base de datos del proyecto.
